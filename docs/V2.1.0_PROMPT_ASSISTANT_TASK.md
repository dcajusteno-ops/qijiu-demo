# v2.1.0 提示词提示器任务书

## 1. 任务背景

`Comfy Manager` 在本任务立项时的稳定版本为 `v2.0.1`，当前正式发布版本已更新为 `v2.1.0`。项目原有的提示词相关基础能力如下：

- 已有“提示词模板”功能
- 已有“提示词工具链接”功能
- `Lightbox` 可读取图片元数据中的 `Prompt / Positive / Negative`
- 前端已有提示词相关组件基础：
  - `desktop-source/frontend/src/components/PromptTemplateDialog.vue`
  - `desktop-source/frontend/src/components/PromptToolsDropdown.vue`
- 后端已有本地持久化文件：
  - `data/prompt-templates.json`
  - `data/prompt-tool-links.json`

因此，`v2.1.0` 的“提示词提示器”不应是孤立新页，而应成为现有图库、图片详情、模板库之间的提示词工作台。

---

## 2. 本次结论先说清

### 2.1 原始数据是否损坏

结论：`提示词/downloaded_prompt_data/all_prompts_merged.json` 本身是可正常解析的 `UTF-8 JSON`，并非结构性损坏。

之前看到的乱码，主要来自两个问题：

- Windows 终端/PowerShell 的显示编码导致阅读时出现乱码假象
- 原始整合数据中确实存在少量字段不规整，例如首尾空格、空子分类、重复记录

### 2.2 运行时是否直接读取 `提示词` 目录

结论：**不要直接读取** `提示词/downloaded_prompt_data/` 下的原始整合文件作为运行时数据源。

原因：

- 该目录更适合作为抓取/导出中间产物
- 不利于后续版本做数据缓存、清洗和回滚
- 与程序自己的 `data/` 持久化目录职责不一致

### 2.3 当前已完成的数据修复

已新增项目内部可用副本：

- `data/prompt-library/all_prompts_merged.cleaned.json`
- `data/prompt-library/manifest.cleaned.json`

已新增可重复执行的清洗脚本：

- `tools/build_prompt_library.py`

当前这份清洗副本的处理结果：

- 原始记录数：`17253`
- 清洗后记录数：`17222`
- 去除精确重复：`31`
- `text_en` 首尾空格修复：`173`
- `text_zh` 首尾空格修复：`41`
- `subcategory` 空值补全为空字符串：`146`
- 落盘编码：`UTF-8`

结论：`v2.1.0` 后续功能应统一读取 `data/prompt-library/all_prompts_merged.cleaned.json`。

## 3. 版本目标

### 3.1 核心目标

围绕项目内部清洗后的提示词词库，做一个本地离线可用的“提示词提示器”，满足以下能力：

- 搜索提示词
- 按来源/分类/子分类/作用域筛选
- 将词条加入正向或反向 Prompt
- 一键复制
- 一键保存到模板库
- 从图库页和图片详情页直接打开

### 3.2 版本定位

`v2.1.0` 的目标不是替代外部提示词网站，而是补齐 `Comfy Manager` 内部的提示词工作流闭环：

- 看图
- 找词
- 拼 Prompt
- 保存模板
- 继续回图或出图

## 4. 数据策略

### 4.1 数据源分层

建议采用双层结构：

#### 原始源目录

- `提示词/downloaded_prompt_data/`

职责：

- 保存抓取结果
- 作为离线原始来源
- 不直接参与运行时读取

#### 运行时目录

- `data/prompt-library/`

职责：

- 保存清洗后的稳定副本
- 作为程序运行时唯一读取入口
- 保存清洗清单与版本信息

### 4.2 运行时正式数据文件

本次建议固定为：

- `data/prompt-library/all_prompts_merged.cleaned.json`

配套清单：

- `data/prompt-library/manifest.cleaned.json`

### 4.3 清洗规则

当前清洗流程至少包含：

- 统一按 `UTF-8` 读取
- 清理首尾空格
- 空字段补默认空字符串
- 生成稳定 `id`
- 生成统一搜索字段 `search_text`
- 去除精确重复项
- 落盘为 `UTF-8` 中文可读 JSON

### 4.4 后续更新方式

如果后续重新抓取词库，建议统一执行：

```bash
python tools/build_prompt_library.py
```

不要手工复制覆盖运行时文件。

## 5. 提示词提示器功能定义

### 5.1 功能名称

`提示词提示器`

### 5.2 功能说明

提示词提示器是一个本地词库检索与 Prompt 拼装工具，数据来源于清洗后的项目内部词库副本，而不是外部网站实时请求。

它应提供：

- 词条搜索
- 词条筛选
- 词条预览
- 正向/反向拼装
- 复制结果
- 保存模板
- 与图片元数据联动

---

## 6. UI 风格要求

### 6.1 视觉方向

本次 UI 明确采用：**简约风**

要求：

- 结构清晰
- 少装饰
- 少视觉噪音
- 优先突出检索、筛选、拼装效率

### 6.2 设计原则

- 不做厚重卡片堆叠
- 不做花哨渐变和强装饰背景
- 不做复杂动效
- 控件层级要轻
- 信息密度适中，优先可扫读

### 6.3 建议风格

- 浅色中性底色为主
- 低饱和边框与分隔
- 关键词使用少量强调色
- 输入框、筛选项、拼装区保持统一组件风格
- 保持与当前项目现有 shadcn-vue / Tailwind 风格协调

### 6.4 文本与编码要求

本次功能的文本必须支持：

- `UTF-8`
- 中文
- 英文

要求：

- 前端文案文件必须以 `UTF-8` 保存
- 运行时词库数据必须以 `UTF-8` 保存
- 不允许再次把终端乱码误判为数据乱码
- 组件内显示中文时必须以真实字段内容为准，不以控制台显示效果为准

## 14. 风险与注意事项

### 14.1 主要风险

- 后续重新抓取的原始数据可能再次带入重复项或空字段
- 终端输出乱码可能误导排查方向
- 如果运行时直接依赖原始目录，会让后续升级和回滚困难

### 14.2 开发注意点

- 运行时统一读取 `data/prompt-library/`
- 不要把原始抓取目录当作正式业务数据目录
- 全链路坚持 `UTF-8`
- 中文显示问题优先核对文件编码和前端渲染，不要先怀疑数据本身损坏
