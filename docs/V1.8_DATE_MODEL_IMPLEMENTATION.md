# v1.8 日期产出工作台与模型 / LoRA 筛选实现文档

## 概述

本文档用于把 [V1.8_DATE_MODEL_PLAN.md](D:/AiImg/ComfyUI-aki-v3/ComfyUI/output/comfy-manager/docs/V1.8_DATE_MODEL_PLAN.md) 落到可直接开发的实现层。

本次实现只围绕两条主线展开：

1. 日期产出工作台
2. 模型 / LoRA 筛选

核心原则：

- 不改用户现有 `ComfyUI/output` 目录习惯
- 复用现有图库页，不做第二套图片浏览系统
- UI 保持简约风，减少“功能感”，强化“工作台感”

---

## 一、UI 方向

## 1. 视觉原则

简约风不是“空”，而是“信息有层次，控件克制”。

本版 UI 建议遵循：

- 使用现有圆角卡片语言：`rounded-[24px]` 到 `rounded-[30px]`
- 使用低对比边框：`border-border/70` 或 `border-border/80`
- 背景以 `bg-card` 和 `bg-background` 为主，不增加强色块大面板
- 强调信息分区，不强调装饰
- 统计卡片数量控制在 3 到 4 个
- 顶部筛选只保留最必要控件，不做过多按钮栏
- 颜色重点用于状态和选中，不用于大面积视觉冲击

## 2. 禁止事项

本版不建议：

- 做成“数据大屏”
- 使用密集图表和复杂渐变
- 加太多筛选抽屉和弹层
- 在一个页面堆太多快捷入口
- 让模型 / LoRA 筛选变成重配置面板

## 3. 推荐页面气质

可以参考当前项目中较贴近的页面气质：

- [ProfileCenter.vue](D:/AiImg/ComfyUI-aki-v3/ComfyUI/output/comfy-manager/desktop-source/frontend/src/components/ProfileCenter.vue)
- [StatisticsDashboard.vue](D:/AiImg/ComfyUI-aki-v3/ComfyUI/output/comfy-manager/desktop-source/frontend/src/components/StatisticsDashboard.vue)

但要比统计页更轻，比个人中心更工具化。

## 二、功能落点

## 1. 日期产出工作台

### 入口建议

新增一个主入口：

- `date-workbench`

建议在以下位置之一接入：

- 侧栏主入口，靠近 `dashboard` 和 `statistics`
- 或首页工作台中的快捷入口卡片

推荐优先做成侧栏一级入口，权重足够清晰。

### 页面结构

建议新建组件：

- `DateWorkbench.vue`

页面结构：

1. 顶部摘要区
2. 快捷时间切换区
3. 指定日期区
4. 当前结果摘要区
5. 最近日期列表区

### 顶部摘要区

建议 4 张卡片：

- 今日出图
- 昨日出图
- 最近 7 天
- 本月出图

卡片风格：

- 高度统一
- 仅一行标题 + 一行大数字 + 一行说明
- 图标小而克制

### 快捷时间切换区

建议做成一排轻量按钮：

- 今天
- 昨天
- 最近 7 天
- 本月
- 全部日期

交互要求：

- 选中状态清晰
- 点击即更新图库结果
- 不弹窗

### 指定日期区

建议只保留一个日期选择器和一个清除按钮：

- 日期选择器
- 清除指定日期

如果用户选了指定日期，则优先级高于快捷时间按钮。

### 当前结果摘要区

建议展示：

- 当前筛选名称
- 命中图片数
- 命中日期数
- 一个“在图库中查看”按钮

### 最近日期列表区

建议做成简洁列表，不做日历大面板。

每行显示：

- 日期
- 当天出图数
- 最近更新时间

点击一行后直接切换到该日期结果。

## 三、模型 / LoRA 筛选 UI

## 1. 接入位置

不要单独做页面。

推荐接入现有图库页顶部筛选区，做成两个轻量筛选器：

- 模型
- LoRA

建议新增组件：

- `ModelFilterBar.vue`

也可以拆成：

- `ModelFilterSelect.vue`
- `LoraFilterSelect.vue`

但如果范围控制优先，建议先做一个合并筛选栏组件。

## 2. 推荐布局

在图库顶部搜索框附近新增：

- 模型下拉
- LoRA 下拉
- 当前筛选摘要
- 清空筛选

布局要求：

- 一行内优先展示
- 宽度不足时自动换行
- 不遮挡现有搜索与排序

## 3. 模型筛选器

建议展示逻辑：

- 默认显示“全部模型”
- 下拉中展示模型名 + 使用次数
- 长模型名截断显示
- hover 时显示完整名称

## 4. LoRA 筛选器

建议展示逻辑：

- 默认显示“全部 LoRA”
- 下拉中展示 LoRA 名 + 使用次数
- 名称过长时截断
- 保留原始匹配值

## 5. 当前筛选摘要

建议做成轻量标签区：

- 日期：最近 7 天
- 模型：PonyXL
- LoRA：角色增强

每个摘要标签都可单独清除。

## 2. useImages.js 增强

建议新增状态：

- `activeDateValue`
- `activeModelFilter`
- `activeLoraFilter`

建议新增计算逻辑：

3. 模型过滤

这样改动最小，也最稳定。

- `extractDateFolder(relPath)`

- `getDatePresetRange(preset)`
- `matchImageByDatePreset(image, preset)`

## 五、后端实现方案

## 1. 目标

后端尽量只做数据补强，不额外引入新存储。

建议优先复用：

- `ImageMetaCacheEntry.Model`
- `ImageMetaCacheEntry.Loras`
- 已有统计接口

## 2. 推荐新增接口

如果前端直接聚合性能可接受，则可不新增接口。

如果需要减轻前端压力，建议新增：

- `GetModelUsageStats()`
- `GetLoraUsageStats()`
- `GetDateWorkbenchSummary()`

返回内容建议尽量轻量，只返回聚合结果，不返回整图数据。

## 3. 推荐修改文件

- `desktop-source/app.go`

关注点：

- metadata 覆盖率
- 模型名和 LoRA 名提取稳定性
- 聚合统计性能

## 六、状态与导航设计

## 1. 新 root 建议

建议新增：

- `date-workbench`

在 `App.vue` 中与现有：

- `dashboard`
- `statistics`
- `profile`

保持同级切换。

## 2. 页面间联动

推荐行为：

- 在日期工作台点击某个时间条件后，更新筛选状态
- 如果用户点击“查看图库”，则切到图库 root
- 但筛选状态保留

这样既有工作台体验，也有图库结果承接。

## 七、简约风 UI 细节约束

## 1. 组件样式

建议统一：

- 输入框高度：`h-11`
- 卡片圆角：`rounded-[24px]` 或 `rounded-[30px]`
- 按钮圆角：`rounded-2xl`
- 卡片阴影：`shadow-none`

## 2. 图标使用

建议日期工作台只使用少量图标：

- `CalendarDays`
- `Clock3`
- `Sparkles`
- `TrendingUp`

模型 / LoRA 筛选建议只用：

- `Box`
- `Tag`

避免图标过多造成视觉噪音。

## 3. 文案风格

建议文案短句化：

- 今日出图
- 最近 7 天
- 全部模型
- 全部 LoRA
- 清空筛选

不要出现长段功能说明常驻页面。

## 4. 空状态

空状态建议简洁：

- 这段时间还没有图片
- 当前模型下没有命中结果
- 当前 LoRA 下没有命中结果

## 八、开发顺序

## 第一阶段：数据准备

1. 检查 metadata 中模型 / LoRA 覆盖率
2. 在 `useImages.js` 中加入聚合逻辑
3. 完成日期目录识别工具函数

## 第二阶段：筛选链路

1. 接入日期过滤
2. 接入模型过滤
3. 接入 LoRA 过滤
4. 验证与现有标签 / 搜索叠加逻辑

## 第三阶段：页面与 UI

1. 新建 `DateWorkbench.vue`
2. 新建 `ModelFilterBar.vue`
3. 接入侧栏与主视图切换
4. 接入图库顶部筛选

## 第四阶段：打磨

1. 当前筛选摘要
2. 清空筛选
3. 空状态
4. 长名称显示
5. 分页与结果数量校正
