# v1.8 日期产出工作台与模型 / LoRA 筛选规划

## 概述

v1.8 只聚焦两条主线：

1. 日期产出工作台
2. 按模型 / LoRA 筛选

这一版的目标不是增加复杂配置，而是让现有 `ComfyUI/output` 目录在不改变使用习惯的前提下，变得更容易浏览、筛选和回看。

---

## 一、版本目标

### 1. 核心目标

- 不要求用户调整现有 `ComfyUI/output` 目录结构
- 直接支持 `2026-04-07` 这类日期文件夹产出
- 让用户能快速查看今天、最近几天、本月的出图
- 让用户能快速找到“某个模型出的图”或“某个 LoRA 用过的图”
- 所有筛选结果直接复用现有图库，不制造第二套重复页面逻辑

### 2. 非目标

本版本暂不优先处理：

- 复杂目录规则系统
- 多层级自定义工作区配置
- 高级搜索语法
- 大规模自动整理 / 自动迁移文件
- 相似图折叠 / 批量筛图模式

---

## 二、功能范围

## 1. 日期产出工作台

### 功能目标

为 ComfyUI 常见日期目录提供一个专用入口，替代用户手动翻文件夹。

### 必须包含

- 自动识别 `YYYY-MM-DD` 格式目录
- 提供以下快捷视图：
  - `今天`
  - `昨天`
  - `最近7天`
  - `本月`
  - `指定日期`
- 进入某个日期视图后，结果直接联动现有图库页
- 显示当前日期视图命中的图片数量
- 没有命中时提供清晰空状态

### 应该包含

- 今日出图数
- 本周出图数
- 本月出图数
- 最近活跃日期
- 日期列表按最新日期优先展示

### 可以以后做

- 日期热力图
- 日期趋势图
- 周视图 / 月视图切换
- “今天 vs 昨天”对比摘要

## 2. 模型筛选

### 功能目标

基于图片 metadata 中已有的模型字段，提供稳定可用的模型过滤能力。

### 必须包含

- 自动聚合所有已解析模型名称
- 提供模型筛选入口
- 支持单模型筛选
- 模型筛选结果与日期筛选可叠加
- 无模型 metadata 的图片不报错，只是不参与模型命中

### 应该包含

- 模型使用次数统计
- 最近常用模型列表
- 模型名称去重与统一展示
- 结果列表显示当前筛选模型

### 可以以后做

- 模型排行榜
- 模型使用趋势
- 模型收藏 / 常用置顶

## 3. LoRA 筛选

### 功能目标

把图片 metadata 中的 LoRA 列表变成真正可点、可筛、可统计的数据入口。

### 必须包含

- 自动聚合所有 LoRA 名称
- 支持单个 LoRA 筛选
- LoRA 筛选与日期筛选可叠加
- LoRA 筛选与模型筛选可叠加
- 无 LoRA 的图片不报错

### 应该包含

- LoRA 使用次数统计
- 热门 LoRA 列表
- 对过长 LoRA 名称做可读展示
- 保留原始名称以便精确命中

### 可以以后做

- 多 LoRA 同时筛选
- LoRA 分组
- LoRA 使用趋势

## 三、推荐交互方案

## 1. 新入口结构

推荐在现有主界面中新增一个独立入口：

- `日期产出`

该入口进入后显示：

- 顶部统计卡片
- 快捷时间过滤按钮
- 日期列表或日期摘要
- 当前结果直接映射到图库

模型 / LoRA 筛选不建议单独做新页面，建议作为全局侧边筛选区或图库顶部筛选区存在。

## 2. 推荐页面结构

### 日期产出工作台

- 顶部：
  - 今日出图
  - 最近7天
  - 本月
  - 最近活跃日期
- 中部：
  - 快捷时间切换按钮
  - 指定日期选择器
- 下部：
  - 当前日期范围结果预览
  - 点击后进入图库结果

### 图库筛选增强

在现有图库页新增：

- 模型筛选下拉 / 面板
- LoRA 筛选下拉 / 面板
- 当前筛选条件摘要
- 一键清空筛选

## 四、数据与实现建议

## 1. 日期识别来源

优先基于图片 `relPath` 的顶层或上层目录识别日期目录。

推荐规则：

- 识别目录名是否匹配 `^\d{4}-\d{2}-\d{2}$`
- 将该目录视为图片所属产出日期
- 不依赖用户手工打标签

## 2. 模型 / LoRA 数据来源

直接复用现有 metadata 缓存：

- `ImageMetaCacheEntry.Model`
- `ImageMetaCacheEntry.Loras`

不建议为 v1.8 单独设计新的数据库或新缓存文件。

## 3. 前端状态建议

建议新增以下筛选状态：

- `activeDatePreset`
- `activeDateValue`
- `activeModelFilter`
- `activeLoraFilter`

并统一接入 `useImages.js` 的最终过滤链路。

## 4. 联动原则

最终图片集合建议按以下顺序过滤：

1. 当前目录 / 当前视图
2. 日期筛选
3. 模型筛选
4. LoRA 筛选
5. 现有标签 / 搜索 / 尺寸等过滤条件

这样可以最大程度复用现有逻辑，减少回归风险。

## 六、验收标准

### 日期产出工作台

- 能正确识别 `YYYY-MM-DD` 日期目录
- 点击 `今天` 能正确显示今天目录下的图片
- 点击 `最近7天` 能正确汇总最近 7 天图片
- 指定日期后结果正确联动图库
- 没有对应日期图片时空状态正确

### 模型筛选

- 能从 metadata 中正确提取模型
- 点击某模型后只显示该模型图片
- 模型筛选与日期筛选叠加后结果正确
- 清空模型筛选后结果恢复正常

### LoRA 筛选

- 能从 metadata 中正确提取 LoRA
- 点击某 LoRA 后只显示命中图片
- LoRA 与模型、日期筛选叠加后结果正确
- 无 LoRA 的图片不会导致异常

## 七、推荐开发顺序

1. 先补齐 metadata 数据完整性
2. 再实现模型 / LoRA 聚合与筛选
3. 然后实现日期目录识别与快捷时间视图
4. 再做日期产出工作台 UI
5. 最后补统计卡片、筛选摘要和空状态
